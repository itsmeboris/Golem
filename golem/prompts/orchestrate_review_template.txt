## Stage 2 - Code Quality Review

Only after Spec Review passes. Dispatch a Quality Reviewer (subagent_type: "reviewer") with:

Working directory: {work_dir}

### What was built
[Files changed, design decisions, spec statements]

### Your task
Review all changed files for bugs, edge cases, naming clarity,
test quality, code duplication, and security issues. Pay special attention
to these high-value bug classes:

1. **Concurrency and locking**: File locks must span the entire
   read-modify-write cycle. Locks on temp files or partial operations
   leave race windows.
2. **Async / executor exception handling**: `loop.run_in_executor()`
   returns a Future - if not awaited, exceptions inside the callable are
   silently dropped. `try/except` must be inside the callable, not
   around the scheduling call.
3. **Ordering-dependent deduplication**: When items are deduplicated via
   overlap/similarity, verify that insertion order does not suppress
   legitimate entries (e.g., adding a superset before its parts).
4. **Exception path test coverage**: Verify that `except` branches are
   exercised by tests, not just the happy path.
5. **Weak test assertions**: Tests that achieve coverage without meaningful
   verification - tautological assertions (`assert f(x) == f(x)`),
   assertion-free test functions, or tests where all assertions are
   against mock return values rather than computed output.
6. **Classification overlap / partition invariants**: When code classifies
   input into exclusive categories (fractions, enum branches, tagged
   unions), verify that (a) categories are mutually exclusive — the same
   input region is not counted in two buckets, (b) fractions/percentages
   sum to the expected total (≤1.0, ==100%, etc.), and (c) edge inputs
   that straddle boundaries are tested (e.g., non-ASCII characters
   inside code blocks, items matching multiple regex patterns).
7. **Boundary / empty-input behavior**: For each new function, check what
   happens with empty input, single-element input, and input at the
   exact boundary of any threshold or limit. Verify these are tested.

Only report issues with confidence >= 80. Read each file once - do not
re-read or run tests.

Report: APPROVED or NEEDS_FIXES: [confidence, file:line, fix suggestion]
