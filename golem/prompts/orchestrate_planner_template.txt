# Planner Dispatch Template

Use this template when the orchestrator dispatches a Planner subagent during
the PLAN phase. The Planner explores the codebase and produces a structured
implementation plan. It does NOT modify any files.

---

## Implementation Steps

Break the work into bite-sized steps of 2–5 minutes each. Each step MUST:

- Use checkbox syntax (`- [ ]`) for tracking
- Follow TDD order: write the failing test first, then implement, then verify
- Include exact code — no placeholders, no pseudocode

### Example step format

- [ ] **Step 1 — Add `FrobResult` TypedDict to `golem/types.py`**

  ```python
  # In golem/types.py, after the existing TypedDicts:
  class FrobResult(TypedDict):
      value: int
      label: str
  ```

  Verify: `pylint --errors-only golem/types.py` passes.

- [ ] **Step 2 — Write failing test for `frobnicate()` in `golem/tests/test_example.py`**

  ```python
  def test_frobnicate_returns_labeled_result(self):
      result = frobnicate(42)
      assert result["value"] == 42
      assert result["label"] == "answer"
  ```

  Run: `pytest golem/tests/test_example.py::TestExample::test_frobnicate_returns_labeled_result -x`
  Expected: **FAIL** (function does not exist yet).

- [ ] **Step 3 — Implement `frobnicate()` in `golem/example.py`**

  ```python
  from golem.types import FrobResult

  def frobnicate(n: int) -> FrobResult:
      """Return a labeled result for n."""
      return FrobResult(value=n, label="answer")
  ```

  Run: `pytest golem/tests/test_example.py -x`
  Expected: **PASS**.

## No Placeholder Rule

The plan MUST NOT contain any of the following. Reject the plan and revise if
any appear:

| Forbidden phrase | Why it is forbidden |
|---|---|
| "TBD" | Leaves work undefined; the Builder will guess |
| "TODO" | Same as TBD — no deferred decisions in a plan |
| "implement later" | Deferred work silently drops from scope |
| "Add appropriate error handling" | What exception? What response? Be exact |
| "Write tests for the above" | Which cases? Parametrized or not? Be exact |
| "Similar to Task N" | Repeat the actual code — do not cross-reference |
| Steps without code blocks | Every step requires exact code, not prose |

# Plan Reviewer Criteria

The orchestrator may dispatch a Reviewer subagent to validate the plan before
handing it to Builders. The Reviewer checks the following criteria:

| Criterion | What to verify |
|---|---|
| **Completeness** | Every file mentioned in the task description appears in the File Map |
| **No placeholders** | None of the forbidden phrases from the No Placeholder Rule appear |
| **Type consistency** | All new TypedDicts are added to `golem/types.py`; no inline definitions |
| **Buildability** | Steps are in dependency order — no step references something created later |
| **File coverage** | Every file in the File Map appears in at least one Implementation Step |
| **TDD order** | Each feature step is preceded by a failing-test step |
| **Code exactness** | Every step contains real, importable Python — no pseudocode |

Report: **APPROVED** or **NEEDS_REVISION**: [criterion, issue, suggested fix]

Only report issues with confidence >= 80. Minor style preferences should be skipped.
